LTspice — Module.

Main module for LTspice.jl - a Julia interface to LTspice

LTspice.LTspiceSimulation — Type.

Access parameters and measurements of an LTspice simulation. Runs simulation as needed.

Access as a dictionary:

measurement_value = sim["measurement_name"]
parameter_value = sim["parameter_name"]
sim["parameter_name"] = new_parameter_value

Access as a function:

(m1,m2,m3) = sim(p1,p2,p3)  # simulation with three measurements and three parameters

Access as arrays or tuples:

pnames = parameternames(sim)
mnames = measurementnames(sim)
snames = stepnames(sim)
pvalues = parametervalues(sim)
mvalues = measurementvalues(sim)
svalues = stepvalues(sim)

Retruns measurements of sim as an a array of Float64 values.

Base.flush — Function.
flush(sim)

Writes sim's circuit file back to disk if any parameters have changed. The user does not usually need to call flush. It will be called automatically when a measurement is requested and the log file needs to be updated. It can be used to update a circuit file using julia for simulation with the LTspice GUI.

source
LTspice.run! — Function.
run!(sim)

Writes circuit changes, calls LTspice to run sim, and reloads the log file. The user normally does not need to call this.

Exporting Data

LTspice.perlineiterator — Function.
perlineiterator(simulation, <keyword arguments>)

Retruns iterator that flattens multidimensional data.

Keyword Arguments

  • steporder – specify order of steps
  • resultnames – specify parameters and measurements for output
  • header – true to make first line header

The step order defaults to the order the step values appear in the circuit file. Step order can be specified by passing an array of step names. By default there is one column for each step, measurement, and parameter. The desired measurements and parameters can be set by passing an array of names to resultnames.

# write CSV with headers
using DelimitedFiles
writedlm("test.csv",perlineiterator(circuit2,header=true),',')
